# ADR-001: Telegram как платформа чат-бота

**Дата:** 25.09.2024
**Статус:** Accepted
**Автор:** Петрова П. П. (РП)
**Связанные решения:** ADR-006

---

## Контекст

На старте проекта команда выбирала платформу для размещения
чат-бота-консультанта. Ключевые требования: доступность для студентов
без установки дополнительного ПО, скорость разработки MVP,
минимальные операционные расходы.

---

## Рассматриваемые варианты

**Вариант 1: Telegram Bot API**
+ Массовое использование в студенческой среде НИЯУ МИФИ.
+ Бесплатный Bot API с богатой документацией.
+ Быстрый прототип: MVP за 2–3 недели.
+ Нативная поддержка файлов, изображений, кнопок.
− Ограничения платформы: нельзя инициировать диалог без подписки.
− Зависимость от внешнего сервиса.

**Вариант 2: VK Mini Apps**
+ Интеграция с VK — популярной соцсетью среди студентов.
− Требует разработки фронтенда (HTML/JS/CSS).
− Сложнее в поддержке, нет готовых библиотек для Python.
− Менее удобен для быстрых текстовых ответов и шаблонов.

**Вариант 3: Собственное веб-приложение**
+ Полный контроль над функциональностью и дизайном.
− Требует разработки фронтенда, бэкенда и инфраструктуры.
− Студенты должны переходить по ссылке — дополнительный барьер входа.
− Срок разработки MVP вырастает до 2–3 месяцев.

---

## Решение

Выбран **Telegram Bot API**. Фактор, предрешивший выбор: опрос показал,
что 100% студентов ВИШ пользуются Telegram ежедневно. Скорость
прототипирования и бесплатность платформы — дополнительные аргументы.

---

## Последствия

- Разработка ведётся на Python с библиотекой `python-telegram-bot`.
- Функциональность ограничена возможностями Telegram Bot API
  (нет произвольного UI, нет проверки личности пользователя).
- Зависимость от доступности серверов Telegram.
- Переход на другую платформу потребует переписывания уровня
  взаимодействия с пользователем.
