# ADR-006: Кнопочная навигация vs семантический поиск

**Дата первичного решения:** 16.10.2024
**Дата пересмотра:** 27.02.2025
**Статус:** Accepted (открытый вопрос для следующей итерации)
**Автор:** Петрова П. П. (РП), Смирнов С. А. (аналитик)
**Связанные решения:** ADR-001

---

## Контекст

На старте проекта (октябрь 2024) команда выбрала кнопочную навигацию
как основной UX-паттерн — быстро, предсказуемо, не требует NLP.
В феврале 2025 года фокус-группа (первая итерация опросов) принесла
однозначный сигнал: пользователи хотят набирать вопрос текстом,
а не листать меню. Решение потребовало пересмотра.

---

## Рассматриваемые варианты

**Вариант 1: Оставить кнопочную навигацию**
+ Реализовано, стабильно работает.
+ Контент заранее структурирован — пользователь не может «потеряться».
− Неудобно при глубокой иерархии разделов.
− Не масштабируется: при росте числа разделов меню перегружается.

**Вариант 2: Поиск по ключевым словам**
+ Простая реализация: полнотекстовый поиск по PostgreSQL.
+ Не требует внешних API.
− Плохо работает при опечатках и синонимах.
− Не понимает намерения: «как получить деньги» ≠ «стипендия».

**Вариант 3: Семантический поиск с LLM**
+ Понимает смысл вопроса независимо от формулировки.
+ Масштабируется без реструктуризации меню.
− Требует интеграции с внешним API (OpenAI и др.) или
  развёртывания собственной модели.
− Дополнительные расходы и зависимость от внешнего сервиса.
− Выходит за рамки текущего стека команды.

---

## Решение

В текущей итерации **оставить кнопочную навигацию** как основной
интерфейс. Ограничения по срокам (защита в мае 2025) и стеку
команды не позволяют реализовать семантический поиск качественно.

Решение принято как временное и задокументировано. Переход к
семантическому поиску рассматривается как следующий этап развития
продукта при наличии ресурсов.

---

## Последствия

- Кнопочная навигация остаётся основным UX до следующей итерации.
- Требование фокус-группы зафиксировано как открытый backlog-item.
- При реализации семантического поиска потребуется: выбор провайдера
  эмбеддингов, настройка векторного хранилища, переработка
  уровня взаимодействия бота с пользователем.
- Потенциальный стек для следующей итерации: Telegram + RAG-агент
  на базе LangChain + pgvector.

---

## Примечание

Данное ADR — прямой прецедент для разработки RAG-агента
как следующей версии системы.
