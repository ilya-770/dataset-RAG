# ADR-003: Миграция контента с JSON на PostgreSQL

**Дата:** 19.02.2025
**Статус:** Accepted
**Автор:** Орлов О. О. (разработчик), Петрова П. П. (РП)
**Отменяет:** ADR-002
**Связанные решения:** ADR-004, ADR-005

---

## Контекст

К началу второго семестра стек проекта потребовал пересмотра.
Накопилось три независимых триггера.

**Рост объёма контента.** За первый семестр аналитики собрали
материал по восьми тематическим разделам; к запланированным добавились
«Поступление» и расширенный блок «АСЭ: трудоустройство». JSON-файлы
стали трудно поддерживаемы: правка одного раздела требовала открытия
нескольких файлов и ручной синхронизации.

**Административная панель.** Заказчик подтвердил требование
к веб-интерфейсу управления контентом без доступа к файлам.
Реализовать это поверх JSON-файлов невозможно без значительных
костылей.

**Смена разработчика.** Орлов О. О. входил в проект с нуля.
Переход к нормальной СУБД упростил онбординг и снизил риск
случайной порчи контента при редактировании вручную.

---

## Рассматриваемые варианты

**Вариант 1: Оставить JSON**
+ Нет затрат на миграцию.
− Не поддерживает административную панель с нормальным UX.
− Нет concurrent-доступа при одновременной работе бота и панели.

**Вариант 2: SQLite**
+ Файловая СУБД, минимальный overhead.
− Ограниченный concurrent write — проблема при одновременных
  обновлениях из панели и чтении ботом.
− Сложнее масштабировать при дальнейшем росте.

**Вариант 3: PostgreSQL**
+ Полноценная СУБД: транзакции, concurrent-доступ, индексы.
+ Хорошо поддерживается в Python-экосистеме (asyncpg, SQLAlchemy).
+ Легко развернуть в Docker-контейнере (официальный образ).
− Требует настройки и первоначальной миграции данных.

---

## Решение

Выбран **PostgreSQL 15**. Данные мигрированы из JSON скриптом
на Python: каждый раздел превратился в запись таблицы `sections`,
кнопки — в `buttons`, ответы — в `responses`. Миграция выполнена
без потери данных за одну рабочую сессию.

---

## Последствия

- Бот и административная панель работают с одной СУБД через
  асинхронный клиент (asyncpg).
- Контент редактируется через веб-панель; прямая правка файлов
  исключена из рабочего процесса.
- Для локального запуска и деплоя PostgreSQL поднимается
  в Docker-контейнере (см. ADR-004).
- Схема БД версионируется через миграции Alembic; откат возможен.
